# 数据库规范

## 1. 使用 Alembic 管理结构变更

所有数据库结构变更必须通过 Alembic Migration 管理，保证不同环境的数据库版本一致。

生成 Migration：

```bash
uv run alembic revision --autogenerate -m "describe schema change"
```

自动生成后必须 Review Migration 内容，再执行升级。

## 2. 禁止直接修改线上表

禁止绕过 Migration 直接修改线上数据库结构。Migration 必须具备清晰的升级和回滚路径，保证变更可追踪、可复现、可回滚。

## 3. 统一命名

数据库表、字段、索引和约束统一使用 `snake_case` 命名。
