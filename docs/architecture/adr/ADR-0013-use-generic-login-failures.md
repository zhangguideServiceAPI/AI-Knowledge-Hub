# ADR-0013 使用统一登录失败响应

## 状态

已接受（2026-07-22）

## 背景

登录需要区分邮箱不存在、密码错误和账号不可用等内部原因，但直接向客户端返回这些细节会帮助攻击者确认某个邮箱是否已经注册。

即使邮箱不存在和密码错误返回相同内容，如果前者直接返回、后者执行较慢的 bcrypt，明显的响应时间差异仍可能形成账号枚举侧信道。

## 决策

- 邮箱不存在和密码错误统一抛出 `InvalidCredentialsError`。
- Router 将两种情况统一转换为 HTTP 401 和 `Invalid email or password.`。
- 401 响应包含 `WWW-Authenticate: Bearer`。
- 邮箱不存在时，仍使用固定 dummy bcrypt hash 执行一次密码验证，再返回失败。
- dummy hash 不是 Secret，不对应真实用户，只用于接近正常密码验证的计算成本。
- 邮箱和密码正确但账号状态不是 `active` 时返回 HTTP 403。
- 不在应用日志中记录原始密码、密码 hash 或完整 Access Token。

## 原因

- 统一错误内容可以避免通过响应正文直接确认账号是否存在。
- dummy bcrypt 校验可以降低未知邮箱和错误密码之间的时间差异。
- 只有凭据正确后才检查并返回账号状态，避免凭据错误时泄露账号状态。
- 401 与 403 分别表达未通过认证和已确认凭据但账号不可用。

## 影响

- 客户端不能根据 401 判断邮箱不存在还是密码错误。
- 未知邮箱请求也会承担一次 bcrypt 计算成本，登录接口后续仍需要限流和防暴力破解能力。
- 测试通过调用行为验证 dummy bcrypt 防护，不使用容易受运行环境影响的精确耗时断言。

## 未采用方案

- 分别返回“邮箱不存在”和“密码错误”：会直接暴露已注册邮箱。
- 未知邮箱立即返回：响应时间可能泄露账号是否存在。
- 对两条失败路径做毫秒级耗时测试：测试易受机器负载和 CI 环境影响，不能稳定证明实现行为。
