# 项目愿景与开发者手册（v1.0）

## 项目使命

通过持续迭代一个真实项目，培养设计、开发、部署、维护 AI 系统的能力。

## 职业目标

AI Systems Engineer → AI Platform Engineer。

## 最终形态

用户、权限、文档、AI Chat、Workflow、RAG、Agent、MCP、监控、CI/CD、Kubernetes。

## 开发原则

先设计后编码；可维护性优先；工程质量高于开发速度。

## AI协作

AI负责重复工作，开发者负责架构和最终决策。

## 学习路线

Sprint-1 项目规范 → Sprint0 基础设施 → Sprint1 认证 → Sprint2 用户与认证会话 → Sprint3 文件 → Sprint4 AI → Sprint5 RAG → Sprint6 Workflow → Sprint7 Agent → Sprint8 MCP → Sprint9 监控 → Sprint10 队列 → Sprint11 k3s。

## 北极星

不是学习框架，而是培养 AI 系统工程能力。
